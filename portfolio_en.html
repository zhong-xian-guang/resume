2. Hall Effect Magnetic Positioning for Mixed Reality Drawing

Master's ThesisVR/MRMachine LearningSensor Hardware

Project Overview

Core research aimed at solving the "occlusion dead-zone failure" problem in traditional optical tracking VR when drawing on physical objects, replacing it with magnetic sensing technology to achieve mixed-reality drawing with no line-of-sight constraints.

🏆 Core Technical Contributions

  • Custom Hardware Array: Manually cut 3D-printed models and hand-soldered signal multiplexers and large arrays of Hall effect sensors across curved surfaces.
  • ML-Driven Cost Reduction & Accuracy: Collected millions of robotic arm Ground Truth data points to train an ML model, successfully compressing average error from 3mm to an impressive 1.2mm — using only 87 sensors instead of the original 321!
  • Smooth 1mm-Accuracy Experience: Applied normal vector computation for smoothing, delivering a near-zero-latency feel. Experimental data confirms the system enables humans to achieve fine model coloring that pure optical VR cannot.

Project Overview

🏆 Core Technical Contributions

  • Hardware-Software Integration & Print Control: Participated in developing the software system for an SLA 3D printer. Used Bluetooth communication to remotely control printer hardware and slice layers, and implemented a cross-platform model printing App management tool.
  • Parametric Architectural Model Generation: Developed a parametric generation engine for miniature architecture (dollhouses), allowing flexible and precise control over interior partition counts, room sizes, and geometry details down to windows, doors, and roofs via parameter inputs.
  • Large Object Smart Segmentation & Interlocking Joint Algorithm: To overcome the machine volume limitation for large wall objects, developed a 3D spatial slicing algorithm that automatically segments large structures and precisely computes and generates 3D interlocking joints on cut surfaces, allowing pieces to be printed by small machines and perfectly assembled.

6. Online Judge (OJ) — Automated Code Grading System

Project ManagementVS Plugin ExtensionAST Semantic Analysis

Project Overview

A large-scale teaching-assistance project led and supervised from end to end. Core goal: upgrade and extend a traditional open-source OJ system to address the educational pain points of modern object-oriented and multi-file project structures.

Core System Evolution

  • Multi-File Compilation & Storage Architecture: Broke through the traditional OJ single-file submission limitation by extending the compiler pipeline and database to support large project submissions containing multiple scripts, enabling true modular testing.
  • Visual Studio IDE Deep Integration: Guided development of a dedicated VS extension, allowing examinees to log in, load assignment projects, submit in stages, and receive real-time compilation feedback — all within a single IDE window.
  • AST Syntax Feature Analysis: To counter students "hardcoding" output to game the grader, integrated static text analysis (AST) to precisely verify whether submitted code actually implements the required classes and functions mandated by each problem.
